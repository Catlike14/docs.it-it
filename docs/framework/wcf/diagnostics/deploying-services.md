## <a name="choosing-the-hosting-environment-for-your-application"></a>Scelta dell'ambiente di hosting per l'applicazione  
 I servizi [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] sono progettati per essere eseguiti in qualsiasi processo di Windows che supporta codice gestito. Per diventare attivo, un servizio deve essere ospitato all'interno di un ambiente di runtime che lo crea e ne controlla il contesto e la durata. Le opzioni di hosting vanno dall'esecuzione in una semplice applicazione console ad ambienti server come un servizio Windows, Internet Information Services (IIS) o all'interno di un processo di lavoro gestito da Windows Activation Service (WAS). Per esaminare le diverse opzioni di hosting per il [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] applicazione, vedere [servizi di Hosting](../../../../docs/framework/wcf/hosting-services.md).
