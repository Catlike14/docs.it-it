# Sviluppo di controlli Windows Form in fase di progettazione
Per gli autori di controlli, .NET Framework offre un'ampia gamma di tecnologie di creazione di controlli. Gli autori non sono più limitati per la progettazione di controlli composti che agiscano come un insieme di controlli preesistenti. Tramite l'ereditarietà, è possibile creare controlli personalizzati da controlli composti preesistenti o controlli Windows Form preesistenti. È inoltre possibile progettare controlli personalizzati che implementano il disegno personalizzato. Queste opzioni consentono una notevole flessibilità per la progettazione e la funzionalità dell'interfaccia visiva. Per sfruttare queste funzionalità, è necessario avere familiarità con i concetti di programmazione basato su oggetti.  
  
> [!NOTE]
>  Non è necessario disporre di una conoscenza approfondita dell'ereditarietà, ma può risultare utile per fare riferimento a [NOT IN BUILD: ereditarietà in Visual Basic](http://msdn.microsoft.com/it-it/e5e6e240-ed31-4657-820c-079b7c79313c).  
  
 Se si desidera creare controlli personalizzati da utilizzare in Web Form, vedere [lo sviluppo di controlli Server ASP.NET](../Topic/Developing%20Custom%20ASP.NET%20Server%20Controls.md).
