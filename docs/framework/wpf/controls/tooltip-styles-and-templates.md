# <a name="tooltip-styles-and-templates"></a>Stili e modelli di ToolTip
In questo argomento vengono descritti gli stili e modelli per il <xref:System.Windows.Controls.ToolTip> controllo. È possibile modificare il valore predefinito <xref:System.Windows.Controls.ControlTemplate> per fornire al controllo un aspetto univoco. Per altre informazioni, vedere [Personalizzazione dell'aspetto di un controllo esistente mediante la creazione di un oggetto ControlTemplate](../../../../docs/framework/wpf/controls/customizing-the-appearance-of-an-existing-control.md).  
  
## <a name="tooltip-parts"></a>Parti della descrizione comando  
 Il <xref:System.Windows.Controls.ToolTip> controllo non dispone di parti denominate.  
  
## <a name="tooltip-states"></a>Stati del controllo ToolTip  
 Nella tabella seguente sono elencati gli stati visivi per la <xref:System.Windows.Controls.ToolTip> controllo.  
  
|Nome VisualState|Nome VisualStateGroup|Descrizione|  
|-|-|-|  
|Chiuso|OpenStates|Lo stato predefinito.|  
|Apri|OpenStates|Il <xref:System.Windows.Controls.ToolTip> è visibile.|  
|Valido|ValidationStates|Il controllo Usa il <xref:System.Windows.Controls.Validation> classe e <xref:System.Windows.Controls.Validation.HasError%2A?displayProperty=nameWithType> proprietà associata è `false`.|  
|InvalidFocused|ValidationStates|Il <xref:System.Windows.Controls.Validation.HasError%2A?displayProperty=nameWithType> proprietà associata è `true` ha il controllo ha lo stato attivo.|  
|InvalidUnfocused|ValidationStates|Il <xref:System.Windows.Controls.Validation.HasError%2A?displayProperty=nameWithType> proprietà associata è `true` ha il controllo non è attivo.|
