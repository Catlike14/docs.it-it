<span data-ttu-id="c9952-103">In questo esempio viene illustrato come implementare un'attività per la quale gli argomenti sono definiti dal consumer dell'attività anziché dall'autore dell'attività.</span><span class="sxs-lookup"><span data-stu-id="c9952-103">This sample demonstrates how to implement an activity for which the arguments are defined by the activity consumer rather than the activity author.</span></span> <span data-ttu-id="c9952-104">Questa operazione viene eseguita tramite l'override del modo in cui i metadati dell'attività vengono costruiti dal runtime.</span><span class="sxs-lookup"><span data-stu-id="c9952-104">It does this by overriding the way the runtime constructs the activity’s metadata.</span></span>

 <span data-ttu-id="c9952-106">Prima dell'esecuzione il runtime compila una descrizione di un'attività esaminando i membri pubblici del tipo di attività e dichiarando automaticamente argomenti, metadata.</span></span> <span data-ttu-id="c9952-107">Ciò ha lo scopo di assicurare la costruzione corretta di un flusso di lavoro, nonché di gestire relazioni della fase di runtime e regole di <xref:System.Activities.Argument>.</span><span class="sxs-lookup"><span data-stu-id="c9952-108">Typically an activity author defines the arguments of an activity by specifying public members on the activity type that derive from <xref:System.Activities.Argument>.</span></span> <span data-ttu-id="c9952-109">Per ogni membro pubblico che deriva da <xref:System.Activities.Argument> il runtime crea public member that derives from <xref:System.Activities.Argument>, the runtime creates a <xref:System.Activities.RuntimeArgument> and binds it to the user-provided argument set on that member.</span></span> <span data-ttu-id="c9952-110">In alcuni casi, tuttavia, il consumer dell'attività fornisce una configurazione che determina il set di argomenti.</span><span data-ttu-id="c9952-111">L'autore di un'attività esegue l'override di <xref:System.Activities.Activity.CacheMetadata%2A> per personalizzare il modo in cui vengono compilati i metadati dell'attività, incluso il set di argomenti associato all'attività.</span><span class="sxs-lookup"><span data-stu-id="c9952-111">An activity author overrides <xref:System.Activities.Activity.CacheMetadata%2A> to  <span data-ttu-id="c9952-112">In questo esempio viene illustrato come compilare dinamicamente un elenco di argomenti per un'attività che richiama un metodo.</span><span class="sxs-lookup"><span data-stu-id="c9952-112">This sample demonstrates how to build an argument list dynamically for an activity that invokes a method.</span></span> <span data-ttu-id="c9952-113">Il consumer dell'attività specifica il tipo e il nome del metodo da richiamare insieme a una raccolta di argomenti da passare a tale metodo.</span><span class="sxs-lookup"><span data-stu-id="c9952-113">The activity consumer specifies the type and the method name they want to invoke along with a collection of arguments to be passed to that method.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="c9952-114">Lo scopo di questo esempio è illustrare come eseguire l'override di <xref:System.Activities.CodeActivity.CacheMetadata%2A> e come usare <xref:System.Activities.RuntimeArgument>.</span><span class="sxs-lookup"><span data-stu-id="c9952-114">The purpose of this sample is to demonstrate how to override <xref:System.Activities.CodeActivity.CacheMetadata%2A> and how to use <xref:System.Activities.RuntimeArgument>.</span></span> <span data-ttu-id="c9952-115">È necessario considerare diversi aspetti per quanto riguarda i tipi di metodi che possono essere richiamati da questa attività.</span><span class="sxs-lookup"><span data-stu-id="c9952-115">There are several caveats with respect to the kinds of methods that this activity can invoke.</span></span> <span data-ttu-id="c9952-116">Ad esempio, non funziona con generics o matrici di parametri.</span><span class="sxs-lookup"><span data-stu-id="c9952-116">For example, it does not work with generics or parameter arrays.</span></span> <span data-ttu-id="c9952-117">L'attività <xref:System.Activities.Statements.InvokeMethod> disponibile in.NET di Framework consente di gestire questi e altri casi.</span><span class="sxs-lookup"><span data-stu-id="c9952-117">The <xref:System.Activities.Statements.InvokeMethod> activity that ships in.NET Framework handles these cases and more.</span></span>  
  
 <span data-ttu-id="c9952-118">L'attività `MethodInvoke` esegue l'override di <xref:System.Activities.CodeActivity.CacheMetadata%2A> e inizia creando un oggetto <xref:System.Activities.RuntimeArgument> per gestire qualsiasi risultato della chiamata del metodo.</span><span class="sxs-lookup"><span data-stu-id="c9952-118">The `MethodInvoke` activity overrides <xref:System.Activities.CodeActivity.CacheMetadata%2A> and begins by creating a <xref:System.Activities.RuntimeArgument> to handle any result from the method invocation.</span></span> <span data-ttu-id="c9952-119">Associa questo <xref:System.Activities.RuntimeArgument> all'oggetto <xref:System.Activities.OutArgument> impostabile pubblicamente denominato `Result`.</span><span class="sxs-lookup"><span data-stu-id="c9952-119">It binds this <xref:System.Activities.RuntimeArgument> to the publicly settable <xref:System.Activities.OutArgument> named `Result`.</span></span> <span data-ttu-id="c9952-120">Se `MethodInvoke.Result` è `null`, il runtime lo popola automaticamente con un oggetto <xref:System.Activities.OutArgument> configurato con l'espressione predefinita per il relativo tipo.</span><span class="sxs-lookup"><span data-stu-id="c9952-120">If `MethodInvoke.Result` is `null`, the runtime automatically populates it with an <xref:System.Activities.OutArgument> configured with the default expression for its type.</span></span> <span data-ttu-id="c9952-121">Questo comportamento significa che l'autore di un'attività non deve mai controllare se la proprietà di un argomento è `null`.</span><span class="sxs-lookup"><span data-stu-id="c9952-121">This behavior means an activity author never has to check whether an argument property is `null`.</span></span>  
  
 <span data-ttu-id="c9952-122">L'override di <xref:System.Activities.CodeActivity.CacheMetadata%2A> determina quindi l'oggetto `MethodInfo` usato per la chiamata dagli oggetti `MethodName` e `TargetType` forniti dall'utente.</span><span class="sxs-lookup"><span data-stu-id="c9952-122">Next, the <xref:System.Activities.CodeActivity.CacheMetadata%2A> override determines the `MethodInfo` it uses for invocation from the user-provided `MethodName` and `TargetType`.</span></span> <span data-ttu-id="c9952-123">Il metodo `DetermineMethodInfo` accetta il parametro <xref:System.Activities.CodeActivityMetadata> passato all'override di <xref:System.Activities.CodeActivity.CacheMetadata%2A> in modo che qualsiasi errore di configurazione possa essere restituito come errore di convalida.</span><span class="sxs-lookup"><span data-stu-id="c9952-123">The `DetermineMethodInfo` method takes the <xref:System.Activities.CodeActivityMetadata> parameter passed to the <xref:System.Activities.CodeActivity.CacheMetadata%2A> override so that any configuration errors can be reported as validation errors.</span></span> <span data-ttu-id="c9952-124">Questa operazione viene effettuata chiamando `metadata.AddValidationError`.</span><span class="sxs-lookup"><span data-stu-id="c9952-124">This is done by calling `metadata.AddValidationError`.</span></span>  
  
 <span data-ttu-id="c9952-125">Dopo l'impostazione di `MethodInfo`, l'esempio scorre i parametri `MethodInfo`.</span><span class="sxs-lookup"><span data-stu-id="c9952-125">Once the `MethodInfo` has been set, the sample iterates over the `MethodInfo` parameters.</span></span> <span data-ttu-id="c9952-126">Per ogni parametro crea un oggetto <xref:System.Activities.RuntimeArgument> e lo associa all'argomento corrispondente nella raccolta fornita dall'utente della proprietà `Parameters`.</span><span class="sxs-lookup"><span data-stu-id="c9952-126">For each parameter, it creates a <xref:System.Activities.RuntimeArgument> and binds it to the corresponding argument in the user-provided collection from the `Parameters` property.</span></span> <span data-ttu-id="c9952-127">Infine, la raccolta di oggetti <xref:System.Activities.RuntimeArgument>s viene associata all'attività mediante una chiamata a `metadata.SetArgumentsCollection`.</span><span class="sxs-lookup"><span data-stu-id="c9952-127">Finally, the collection of <xref:System.Activities.RuntimeArgument>s is associated with the activity by calling `metadata.SetArgumentsCollection`.</span></span>  
  
 <span data-ttu-id="c9952-128">Si noti che la risoluzione dell'argomento può essere effettuata usando un oggetto <xref:System.Activities.RuntimeArgument>, come nel caso di `resultArgument` o l'argomento fornito dall'utente, come nel caso della raccolta `Parameters`.</span><span class="sxs-lookup"><span data-stu-id="c9952-128">Note that argument resolution can be done using a <xref:System.Activities.RuntimeArgument>, as in the case of `resultArgument` or the user-provided argument, as in the case of the `Parameters` collection.</span></span>
