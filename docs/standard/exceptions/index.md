- <span data-ttu-id="431b1-116">La generazione e la gestione delle eccezioni ha lo stesso funzionamento nei linguaggi di programmazione .NET.</span><span class="sxs-lookup"><span data-stu-id="431b1-116">Exception throwing and handling works the same for .NET programming languages.</span></span>

<span data-ttu-id="431b1-120">Le eccezioni presentano vantaggi rispetto ad altri metodi di notifica degli errori, quali i codici restituiti.</span><span class="sxs-lookup"><span data-stu-id="431b1-120">Exceptions offer advantages over other methods of error notification, such as return codes.</span></span> <span data-ttu-id="431b1-121">Gli errori vengono sempre rilevati poiché se viene generata un'eccezione e l'eccezione non viene gestita, il runtime termina l'applicazione.</span><span class="sxs-lookup"><span data-stu-id="431b1-121">Failures do not go unnoticed because if an exception is thrown and you don't handle it, the runtime terminates your application.</span></span> <span data-ttu-id="431b1-122">I valori non validi non continuano a propagarsi nel sistema quando il codice non riesce a verificare la presenza di un codice di errore restituito.</span><span class="sxs-lookup"><span data-stu-id="431b1-122">Invalid values do not continue to propagate through the system as a result of code that fails to check for a failure return code.</span></span>
