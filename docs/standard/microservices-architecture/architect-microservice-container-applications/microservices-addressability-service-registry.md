class="sxs-lookup"><span data-stu-id="b34c1-106">Your microservice needs to be addressable wherever it is running.</span></span> <span data-ttu-id="b34c1-107">Se è necessario che sul computer da cui è in esecuzione un particolare microservizio, operazioni passare errate rapidamente.</span><span class="sxs-lookup"><span data-stu-id="b34c1-107">If you have to think about which computer is running a in modo che sia individuabile posizione corrente del microservizio.</span><span class="sxs-lookup"><span data-stu-id="b34c1-108">In the same way that DNS resolves a URL to a particular computer, your class="sxs-lookup"><span data-stu-id="b34c1-111">In the same vein, when a computer fails, the registry service must be able to indicate where the service is now running.</span></span>

<span data-ttu-id="b34c1-112">Il [modello di servizio Registro di sistema](http://microservices.io/patterns/service-registry.html) è una parte importante dell'individuazione del servizio.</span><span class="sxs-lookup"><span data-stu-id="b34c1-112">The [service registry pattern](http://microservices.io/patterns/service-registry.html) is a key part of service discovery.</span></span> <span data-ttu-id="b34c1-113">Il Registro di sistema è un database contenente i percorsi di rete delle istanze del servizio.</span><span class="sxs-lookup"><span data-stu-id="b34c1-113">The registry is a database containing the network locations of service instances.</span></span> <span data-ttu-id="b34c1-114">Un registro di sistema del servizio deve essere aggiornato e a disponibilità elevata.</span><span class="sxs-lookup"><span data-stu-id="b34c1-114">A service registry needs to be highly available and up to date.</span></span> <span data-ttu-id="b34c1-115">I client è possibile memorizzare nella cache i percorsi di rete ottenuti dal Registro di sistema del servizio.</span><span class="sxs-lookup"><span data-stu-id="b34c1-115">Clients could cache network locations obtained from the service registry.</span></span> <span data-ttu-id="b34c1-116">Tuttavia, tali informazioni alla fine viene aggiornate e i client non è più possono individuare le istanze del servizio.</span><span class="sxs-lookup"><span data-stu-id="b34c1-116">However, that information eventually goes out of date and clients can no longer discover service instances.</span></span> <span data-ttu-id="b34c1-117">Di conseguenza, un registro di sistema del servizio è costituito da un cluster di server che utilizzano un protocollo di replica per mantenere la coerenza.</span><span class="sxs-lookup"><span data-stu-id="b34c1-117">Consequently, a service registry consists of a cluster of servers that use a replication protocol to maintain consistency.</span></span>

<span data-ttu-id="b34c1-118">In alcuni ambienti di distribuzione microservizio (denominati cluster in modo da rientrare in una sezione successiva), l'individuazione del servizio è incorporato.</span><span class="sxs-lookup"><span data-stu-id="b34c1-118">In some microservice deployment environments (called clusters, to be covered in a later section), service discovery is built-in.</span></span> <span data-ttu-id="b34c1-119">Ad esempio, all'interno di un ambiente del servizio di contenitore di Azure, Kubernetes e controller di dominio o del sistema operativo con maratona può gestire la registrazione dell'istanza di servizio e annullamento della registrazione.</span><span class="sxs-lookup"><span data-stu-id="b34c1-119">For example, within an Azure Container Service environment, Kubernetes and DC/OS with Marathon can handle service instance registration and deregistration.</span></span> <span data-ttu-id="b34c1-120">Un proxy vengono anche eseguite in ogni cluster host che svolge il ruolo del router di individuazione sul lato server.</span><span class="sxs-lookup"><span data-stu-id="b34c1-120">They also run a proxy on each cluster host that plays the role of server-side discovery router.</span></span> <span data-ttu-id="b34c1-121">Un altro esempio è Azure Service Fabric, che fornisce inoltre un registro del servizio tramite il servizio di denominazione della casella.</span><span class="sxs-lookup"><span data-stu-id="b34c1-121">Another example is Azure Service Fabric, which also provides a service registry through its out-of-the-box Naming Service.</span></span>

<span data-ttu-id="b34c1-122">Si noti che alcune sovrapposizione tra il Registro di sistema del servizio e il modello di gateway API, che consente di risolvere questo problema anche.</span><span class="sxs-lookup"><span data-stu-id="b34c1-122">Note that there is certain overlap between the service registry and the API gateway pattern, which helps solve this problem as well.</span></span> <span data-ttu-id="b34c1-123">Ad esempio, il [Proxy inverso dell'infrastruttura del servizio](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) è un tipo di implementazione di un Gateway di API che si basa sul servizio Fabrice Naming Service e che consente di risolvere la risoluzione degli indirizzi per i servizi interni.</span><span class="sxs-lookup"><span data-stu-id="b34c1-123">For example, the [Service Fabric Reverse Proxy](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) is a type of implementation of an API Gateway that is based on the Service Fabrice Naming Service and that helps resolve address resolution to the internal services.</span></span>
