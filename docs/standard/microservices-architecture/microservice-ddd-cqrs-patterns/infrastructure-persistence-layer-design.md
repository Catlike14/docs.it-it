## <a name="the-specification-pattern"></a><span data-ttu-id="46d77-183">Schema Specification</span><span class="sxs-lookup"><span data-stu-id="46d77-183">The Specification pattern</span></span>

<span data-ttu-id="46d77-184">Lo schema Specification (il cui nome completo è schema Query-specification) è uno schema progettuale basato su dominio concepito come punto in cui è possibile inserire la definizione di una query con logica facoltativa di ordinamento e paging.</span><span class="sxs-lookup"><span data-stu-id="46d77-184">The Specification pattern (its full name would be Query-specification pattern) is a Domain-Driven Design pattern designed as the place where you can put the definition of a query with optional sorting and paging logic.</span></span>

<span data-ttu-id="46d77-185">Lo schema Specification consente di definire una query in un oggetto.</span><span class="sxs-lookup"><span data-stu-id="46d77-185">The Specification pattern defines a query in an object.</span></span> <span data-ttu-id="46d77-186">Per poter incapsulare una query di paging che cerca alcuni prodotti, ad esempio, è possibile creare una specifica PagedProduct che accetta i parametri di input necessari (pageNumber, pageSize, filter e così via).</span><span class="sxs-lookup"><span data-stu-id="46d77-186">For example, in order to encapsulate a paged query that searches for some products, you can create a PagedProduct specification that takes the necessary input parameters (pageNumber, pageSize, filter, etc.).</span></span> <span data-ttu-id="46d77-187">In questo modo, qualsiasi metodo di repository (in genere un overload List()) accetterà un elemento ISpecification ed eseguirà la query in base a tale specifica.</span><span class="sxs-lookup"><span data-stu-id="46d77-187">Then, within any Repository method (usually a List() overload) it would accept an ISpecification and run the expected query based on that specification.</span></span>

<span data-ttu-id="46d77-188">Esistono diversi vantaggi di questo approccio:</span><span class="sxs-lookup"><span data-stu-id="46d77-188">There are several benefits to this approach:</span></span>

* <span data-ttu-id="46d77-189">La specifica ha un nome (anziché solo una serie di espressioni LINQ) che è possibile usare.</span><span class="sxs-lookup"><span data-stu-id="46d77-189">The specification has a name (as opposed to just a bunch of LINQ expressions) that you can discuss about.</span></span>

* <span data-ttu-id="46d77-190">È possibile eseguire uno unit test della specifica in isolamento per assicurarsi che sia corretta.</span><span class="sxs-lookup"><span data-stu-id="46d77-190">The specification can be unit tested in isolation to ensure it is right.</span></span> <span data-ttu-id="46d77-191">La specifica può anche essere facilmente riusata se è necessario un comportamento simile.</span><span class="sxs-lookup"><span data-stu-id="46d77-191">It can also easily be reused if you need similar behavior.</span></span> <span data-ttu-id="46d77-192">Ad esempio, in un'azione di visualizzazione MVC e in un'azione di API Web, nonché in diversi servizi.</span><span class="sxs-lookup"><span data-stu-id="46d77-192">For example, on an MVC View action and a Web API action, as well as in various services.</span></span>

* <span data-ttu-id="46d77-193">Una specifica può essere usata anche per descrivere la forma dei dati da restituire, in modo che le query possano restituire solo i dati necessari.</span><span class="sxs-lookup"><span data-stu-id="46d77-193">A specification can also be used to describe the shape of the data to be returned, so that queries can return just the data they required.</span></span> <span data-ttu-id="46d77-194">In tal modo si elimina la necessità del caricamento lazy nelle applicazioni Web (che non è in genere consigliabile) consentendo di non congestionare le implementazioni dei repository con questi dettagli.</span><span class="sxs-lookup"><span data-stu-id="46d77-194">This eliminates the need for lazy loading in web applications (which is usually not a good idea) and helps keep repository implementations from becoming cluttered with these details.</span></span>
