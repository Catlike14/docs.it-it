# Convenzioni di denominazione
Segue un set di convenzioni di denominazione nello sviluppo di un framework coerente può essere un importante contributo alla facilità di utilizzo del framework. Consente al framework di essere utilizzato dagli sviluppatori di molti progetti ampiamente separati. Oltre la coerenza del modulo, nomi di elementi framework devono essere facilmente comprensibile e devono indicare la funzione di ogni elemento.  
  
 L'obiettivo di questo capitolo è fornire un set di convenzioni di denominazione coerente che risolve in nomi che ha senso immediato per gli sviluppatori.  
  
 Anche se adottare le convenzioni di denominazione come linee guida sullo sviluppo di codice generale comporterebbe una denominazione più coerente in tutto il codice, è necessario solo per applicarli alle API che sono esposte pubblicamente \(pubblici o protetti tipi e membri e le interfacce implementate in modo esplicito\).
