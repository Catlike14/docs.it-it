## <a name="what-are-layers"></a><span data-ttu-id="26a09-133">Cosa sono i livelli?</span><span class="sxs-lookup"><span data-stu-id="26a09-133">What are layers?</span></span>

<span data-ttu-id="26a09-134">Un modo per gestire la complessità delle applicazioni consiste nel suddividere l'applicazione in base a responsabilità o competenze.</span><span class="sxs-lookup"><span data-stu-id="26a09-134">As applications grow in complexity, one way to manage that complexity is to break the application up according to its responsibilities or concerns.</span></span> <span data-ttu-id="26a09-135">Questo approccio si basa sul principio della separazione delle competenze e può contribuire all'organizzazione di una codebase in crescita consentendo agli sviluppatori di individuare facilmente il punto in cui è implementata una determinata funzionalità.</span><span class="sxs-lookup"><span data-stu-id="26a09-135">This follows the separation of concerns principle, and can help keep a growing codebase organized so that developers can easily find where certain functionality is implemented.</span></span> <span data-ttu-id="26a09-136">Oltre alla semplice organizzazione del codice, l'architettura a livelli offre una serie di vantaggi.</span><span class="sxs-lookup"><span data-stu-id="26a09-136">Layered architecture offers a number of advantages beyond just code organization, though.</span></span>

<span data-ttu-id="26a09-137">L'organizzazione del codice in livelli consente di riusare le funzionalità comuni di basso livello all'interno dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="26a09-137">By organizing code into layers, common low-level functionality can be reused throughout the application.</span></span> <span data-ttu-id="26a09-138">Questa possibilità di riuso è utile poiché consentirà di scrivere una minore quantità di codice e di standardizzare l'applicazione su una singola implementazione, osservando il principio DRY.</span><span class="sxs-lookup"><span data-stu-id="26a09-138">This reuse is beneficial because it means less code needs to be written and because it can allow the application to standardize on a single implementation, following the DRY principle.</span></span>

<span data-ttu-id="26a09-149">I livelli logici sono una tecnica comune per migliorare l'organizzazione del codice nelle applicazioni software aziendali ed esistono molti modi per organizzare il codice in livelli.</span><span class="sxs-lookup"><span data-stu-id="26a09-149">Logical layering is a common technique for improving the organization of code in enterprise software applications, and there are several ways in which code can be organized into layers.</span></span>

> <span data-ttu-id="26a09-150">I *livelli* rappresentano la separazione logica all'interno dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="26a09-150">*Layers* represent logical separation within the application.</span></span> <span data-ttu-id="26a09-151">Nel caso in cui la logica dell'applicazione sia fisicamente distribuita in server o processi separati, queste destinazioni di distribuzione fisiche separate sono denominate *livelli*.</span><span class="sxs-lookup"><span data-stu-id="26a09-151">In the event that application logic is physically distributed to separate servers or processes, these separate physical deployment targets are referred to as *tiers*.</span></span> <span data-ttu-id="26a09-152">È possibile, e piuttosto comune, che un'applicazione a più livelli sia distribuita in un singolo livello.</span><span class="sxs-lookup"><span data-stu-id="26a09-152">It's possible, and quite common, to have an N-Layer application that is deployed to a single tier.</span></span>
