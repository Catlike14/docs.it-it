# Parallel Processing and Concurrency in the .NET Framework
.NET Framework mette a disposizione diversi modi per poter utilizzare più thread di esecuzione, in modo da mantenere l'applicazione reattiva verso l'utente massimizzando nel frattempo le prestazioni del computer dell'utente.  
  
## In questa sezione  
 [Threading](../../docs/standard/threading/index.md)  
 Vengono descritti i meccanismi di base di concorrenza e sincronizzazione forniti da .NET Framework.  
  
 [Asynchronous Programming Patterns](../../docs/standard/asynchronous-programming-patterns/index.md)  
 Viene fornita una breve panoramica dei tre modelli di programmazione asincrona supportati da .NET Framework:  
  
-   [Asynchronous Programming Model \(APM\)](../../docs/standard/asynchronous-programming-patterns/asynchronous-programming-model-apm.md) \(legacy\)  
  
-   [Event\-based Asynchronous Pattern \(EAP\)](../../docs/standard/asynchronous-programming-patterns/event-based-asynchronous-pattern-eap.md) \(legacy\)  
  
-   [Task\-based Asynchronous Pattern \(TAP\)](../../docs/standard/asynchronous-programming-patterns/task-based-asynchronous-pattern-tap.md) \(consigliato per novità sullo sviluppo\)  
  
 [Parallel Programming](../../docs/standard/parallel-programming/index.md)  
 Viene descritto un modello di programmazione basato sulle attività che semplifica lo sviluppo parallelo, consentendo di scrivere codice parallelo efficiente, con granularità fine e scalabile in un linguaggio naturale senza dover utilizzare direttamente i thread o il pool di thread.
