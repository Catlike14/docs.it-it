# <a name="using-threads-and-threading"></a>Utilizzo di thread e threading
Negli argomenti di questa sezione viene illustrato la creazione e gestione dei thread gestiti, come passare i dati e i thread gestiti e ottenere risultati nuovamente e come eliminare i thread e gestire un <xref:System.Threading.ThreadAbortException>.  
  
## <a name="in-this-section"></a>Contenuto della sezione  
 [Creazione di thread e passaggio di dati all'avvio](../../../docs/standard/threading/creating-threads-and-passing-data-at-start-time.md)  
 Viene descritta e illustrata la creazione di thread gestiti, incluso come passare i dati a nuovi thread e come ottenere dati.  
  
 [Sospensione e ripresa di thread](../../../docs/standard/threading/pausing-and-resuming-threads.md)  
 Vengono discusse le implicazioni della sospensione e ripresa di thread gestiti.  
  
 [Eliminazione definitiva di thread](../../../docs/standard/threading/destroying-threads.md)  
 Illustra le conseguenze dell'eliminazione definitiva di thread gestiti e come gestire un <xref:System.Threading.ThreadAbortException>.  
  
 [Pianificazione di thread](../../../docs/standard/threading/scheduling-threads.md)  
 Vengono illustrate le priorità dei thread e gli effetti di pianificazione dei thread.  
  
## <a name="reference"></a>Riferimento  
 <xref:System.Threading.Thread>  
 Fornisce la documentazione di riferimento per il <xref:System.Threading.Thread> (classe), che rappresenta un thread gestito, se proviene da codice non gestito o è stato creato in un'applicazione gestita.  
  
 <xref:System.Threading.ThreadStart>  
 Fornisce la documentazione di riferimento per il <xref:System.Threading.ThreadStart> delegato che rappresenta le routine di thread senza parametri.  
  
 <xref:System.Threading.ParameterizedThreadStart>  
 Fornisce un modo semplice per passare dati a una routine di thread, sebbene senza tipizzazione forte.
