# <a name="entity-references-are-expanded-and-not-preserved"></a><span data-ttu-id="bdbac-102">Riferimenti alle entità espansi e non conservati</span><span class="sxs-lookup"><span data-stu-id="bdbac-102">Entity References are Expanded and Not Preserved</span></span>
<span data-ttu-id="bdbac-103">Quando il riferimento all'entità viene espanso e sostituito dal testo che esso rappresenta, il **XmlEntityReference** nodo non è stato creato.</span><span class="sxs-lookup"><span data-stu-id="bdbac-103">When the entity reference is expanded and replaced by the text it represents, the **XmlEntityReference** node is not created.</span></span> <span data-ttu-id="bdbac-104">Al contrario, la dichiarazione di entità viene analizzata e nodi creati dal contenuto della dichiarazione vengono copiati al posto del **XmlEntityReference**.</span><span class="sxs-lookup"><span data-stu-id="bdbac-104">Instead, the entity declaration is parsed, and nodes created from the content in the declaration are copied in the place of the **XmlEntityReference**.</span></span> <span data-ttu-id="bdbac-105">Pertanto, nel `&publisher;` esempio, il `&publisher;` non viene salvata, ma viene invece un **XmlText** nodo viene creato.</span><span class="sxs-lookup"><span data-stu-id="bdbac-105">Therefore, in the `&publisher;` example, the `&publisher;` is not saved, but instead, an **XmlText** node is created.</span></span>  
  
 <span data-ttu-id="bdbac-106">![struttura ad albero espansa](../../../../docs/standard/data/xml/media/xmlentityref-expanded-nodes.gif "xmlentityref_expanded_nodes")</span><span class="sxs-lookup"><span data-stu-id="bdbac-106">![expanded tree structure](../../../../docs/standard/data/xml/media/xmlentityref-expanded-nodes.gif "xmlentityref_expanded_nodes")</span></span>  
<span data-ttu-id="bdbac-107">Struttura ad albero per i riferimenti alle entità espansi</span><span class="sxs-lookup"><span data-stu-id="bdbac-107">Tree structure for entity references that are expanded</span></span>  
  
 <span data-ttu-id="bdbac-108">Le entità carattere, come `B` o `<`, non sono mantenute,</span><span class="sxs-lookup"><span data-stu-id="bdbac-108">Character entities such as `B` or `<` are not preserved.</span></span> <span data-ttu-id="bdbac-109">ma vengono sempre espanse e rappresentate come nodi di tipo text.</span><span class="sxs-lookup"><span data-stu-id="bdbac-109">Instead, they are always expanded and represented as text nodes.</span></span>  
  
 <span data-ttu-id="bdbac-110">Per mantenere **XmlEntityReference** nodi e i nodi figlio del riferimento all'entità collegata, impostare il **EntityHandling** flag **ExpandCharEntities**.</span><span class="sxs-lookup"><span data-stu-id="bdbac-110">To preserve **XmlEntityReference** nodes and child nodes of the entity reference attached to it, set the **EntityHandling** flag to **ExpandCharEntities**.</span></span> <span data-ttu-id="bdbac-111">In caso contrario, lasciare il **EntityHandling** flag il valore predefinito, ovvero **ExpandEntities**.</span><span class="sxs-lookup"><span data-stu-id="bdbac-111">Otherwise, leave the **EntityHandling** flag at the default, which is to **ExpandEntities**.</span></span> <span data-ttu-id="bdbac-112">In questo caso nel DOM non si osserveranno nodi dei riferimenti alle entità.</span><span class="sxs-lookup"><span data-stu-id="bdbac-112">In this case, you will not see entity reference nodes in the DOM.</span></span> <span data-ttu-id="bdbac-113">I nodi saranno sostituiti da copie dei nodi figlio della dichiarazione di entità.</span><span class="sxs-lookup"><span data-stu-id="bdbac-113">The nodes are replaced by the nodes that are copies of the child nodes of the entity declaration.</span></span>  
  
 <span data-ttu-id="bdbac-114">Una conseguenza della mancata conservazione dei riferimenti alle entità è che quando il documento viene salvato e passato a un'altra applicazione, l'applicazione ricevente non è in grado di rilevare che i nodi sono stati generati da un riferimento all'entità.</span><span class="sxs-lookup"><span data-stu-id="bdbac-114">One side effect of not preserving entity references is that when the document is saved and passed on to another application, the receiving application does not know that the nodes were generated by an entity reference.</span></span> <span data-ttu-id="bdbac-115">Tuttavia, se i riferimenti alle entità vengono conservati, l'applicazione ricevente rileva un riferimento all'entità e legge i nodi figlio.</span><span class="sxs-lookup"><span data-stu-id="bdbac-115">However, when entity references are preserved, a receiving application sees an entity reference and reads the child nodes.</span></span> <span data-ttu-id="bdbac-116">È evidente che i nodi figlio rappresentano le informazioni presenti nella dichiarazione di entità.</span><span class="sxs-lookup"><span data-stu-id="bdbac-116">It is apparent that the child nodes represent the information that was in the entity declaration.</span></span> <span data-ttu-id="bdbac-117">Se i riferimenti alle entità vengono conservati, ad esempio, il DOM possiede teoricamente la struttura seguente.</span><span class="sxs-lookup"><span data-stu-id="bdbac-117">For example, the DOM theoretically has the following structure if entity references are preserved.</span></span>  
  
 <span data-ttu-id="bdbac-118">XmlElement: publisher</span><span class="sxs-lookup"><span data-stu-id="bdbac-118">XmlElement: publisher</span></span>  
  
 <span data-ttu-id="bdbac-119">XmlEntityReference: `&publisher;`</span><span class="sxs-lookup"><span data-stu-id="bdbac-119">XmlEntityReference: `&publisher;`</span></span>  
  
 <span data-ttu-id="bdbac-120">XmlText: Microsoft Press</span><span class="sxs-lookup"><span data-stu-id="bdbac-120">XmlText: Microsoft Press</span></span>  
  
 <span data-ttu-id="bdbac-121">Se i riferimenti alle entità vengono espansi nel DOM, come avviene in base al metodo predefinito, il tipo di struttura ad albero è il seguente:</span><span class="sxs-lookup"><span data-stu-id="bdbac-121">If entity references are expanded in the DOM, which is the default method, the structure has this type of tree:</span></span>  
  
 <span data-ttu-id="bdbac-122">XmlElement: publisher</span><span class="sxs-lookup"><span data-stu-id="bdbac-122">XmlElement: publisher</span></span>  
  
 <span data-ttu-id="bdbac-123">XmlText: Microsoft Press</span><span class="sxs-lookup"><span data-stu-id="bdbac-123">XmlText: Microsoft Press</span></span>  
  
 <span data-ttu-id="bdbac-124">Si noti che il nodo di riferimento di entità è scomparso e l'applicazione ricevente non è possibile indicare che il **XmlText** nodo contenente "Microsoft Press" è stato creato da una dichiarazione di entità.</span><span class="sxs-lookup"><span data-stu-id="bdbac-124">Notice that the entity reference node is gone, and the receiving application cannot tell that the **XmlText** node containing "Microsoft Press" was created from an entity declaration.</span></span>  
  
 <span data-ttu-id="bdbac-125">Se si utilizza un lettore che non è possibile risolvere le entità, il **carico** metodo genera un'eccezione quando viene rilevato un riferimento all'entità.</span><span class="sxs-lookup"><span data-stu-id="bdbac-125">If you use a reader that cannot resolve entities, the **Load** method throws an exception when it encounters an entity reference.</span></span>
