 <span data-ttu-id="cc0d0-175">Come accade con la maggior parte dei modelli a oggetti, alcuni modelli di progettazione funzionano in modo efficiente il `My` dello spazio dei nomi e gli altri non.</span><span class="sxs-lookup"><span data-stu-id="cc0d0-175">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="cc0d0-176">Quando si progetta un'estensione per il `My` dello spazio dei nomi, considerare i principi seguenti:</span><span class="sxs-lookup"><span data-stu-id="cc0d0-176">When designing an extension to the `My` namespace, consider the following principles:</span></span>  
  
-   <span data-ttu-id="cc0d0-177">**Metodi senza stati.**</span><span class="sxs-lookup"><span data-stu-id="cc0d0-177">**Stateless methods.**</span></span> <span data-ttu-id="cc0d0-178">Metodi di `My` dello spazio dei nomi deve fornire una soluzione completa per un'attività specifica.</span><span class="sxs-lookup"><span data-stu-id="cc0d0-178">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="cc0d0-179">Verificare che i valori dei parametri passati al metodo forniscano tutti gli input necessario per completare l'attività particolare.</span><span class="sxs-lookup"><span data-stu-id="cc0d0-179">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="cc0d0-180">Evitare di creare metodi che si basano sullo stato precedente, ad esempio le connessioni aperte alle risorse.</span><span class="sxs-lookup"><span data-stu-id="cc0d0-180">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>
