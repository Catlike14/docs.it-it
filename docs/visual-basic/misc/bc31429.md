 <span data-ttu-id="a0d66-107">Tuttavia, altri linguaggi, ad esempio [c#](../../csharp/csharp.md) e [Visual C++](https://docs.microsoft.com/cpp/top/visual-cpp-in-visual-studio-2015), maiuscole e minuscole.</span><span class="sxs-lookup"><span data-stu-id="a0d66-107">However, other languages, such as [C#](../../csharp/csharp.md) and [Visual C++](https://docs.microsoft.com/cpp/top/visual-cpp-in-visual-studio-2015), are case-sensitive.</span></span> <span data-ttu-id="a0d66-108">In questi linguaggi, `XYZ` e `xyz` non sono considerati uguali.</span><span class="sxs-lookup"><span data-stu-id="a0d66-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="a0d66-109">Di conseguenza, una classe scritta in un linguaggio di questo tipo potrebbe definire una variabile denominata `XYZ` e una proprietà denominata `xyz`.</span><span class="sxs-lookup"><span data-stu-id="a0d66-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="a0d66-110">Common Language Runtime conserva la distinzione tra maiuscole e minuscole negli assembly.</span><span class="sxs-lookup"><span data-stu-id="a0d66-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="a0d66-111">Se però un'applicazione di Visual Basic accede a un assembly con i nomi `XYZ` e `xyz`, i nomi appariranno come uguali.</span><span class="sxs-lookup"><span data-stu-id="a0d66-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>
