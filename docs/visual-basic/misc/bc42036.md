# <a name="operands-of-type-object-used-in-expressions-for-39select39-39case39-statements-runtime-errors-could-occur"></a><span data-ttu-id="9dc90-102">Gli operandi di tipo Object usati nelle espressioni per &#39;selezionare&#39;, &#39;caso&#39; istruzioni; potrebbero verificarsi errori di runtime</span><span class="sxs-lookup"><span data-stu-id="9dc90-102">Operands of type Object used in expressions for &#39;Select&#39;, &#39;Case&#39; statements; runtime errors could occur</span></span>
<span data-ttu-id="9dc90-103">Una costruzione `Select`...`Case` usa una o più espressioni del [Object Data Type](../../visual-basic/language-reference/data-types/object-data-type.md).</span><span class="sxs-lookup"><span data-stu-id="9dc90-103">A `Select`...`Case` construction uses one or more expressions of the [Object Data Type](../../visual-basic/language-reference/data-types/object-data-type.md).</span></span>  
  
 <span data-ttu-id="9dc90-104">Quando una variabile o espressione restituisce `Object`, il compilatore deve eseguire un' *associazione tardiva*, che comporta l'esecuzione di operazioni supplementari in fase di esecuzione.</span><span class="sxs-lookup"><span data-stu-id="9dc90-104">When a variable or expression evaluates to `Object`, the compiler must perform *late binding*, which causes extra operations at run time.</span></span> <span data-ttu-id="9dc90-105">Espone inoltre l'applicazione a possibili errori di runtime.</span><span class="sxs-lookup"><span data-stu-id="9dc90-105">It also exposes your application to potential run-time errors.</span></span> <span data-ttu-id="9dc90-106">Se ad esempio si assegna un oggetto <xref:System.Windows.Forms.Form> a una variabile `Object` e quindi si prova a confrontarla con un numero, viene generata un'eccezione <xref:System.InvalidCastException> perché Visual Basic non supporta la conversione di un oggetto <xref:System.Windows.Forms.Form> in un valore numerico.</span><span class="sxs-lookup"><span data-stu-id="9dc90-106">For example, if you assign a <xref:System.Windows.Forms.Form> to an `Object` variable and then try to compare it with a number, the runtime throws an <xref:System.InvalidCastException> because Visual Basic cannot convert a <xref:System.Windows.Forms.Form> object to a numeric value.</span></span>  
  
 <span data-ttu-id="9dc90-107">Le espressioni contenute in una costruzione `Select`...`Case` devono presentare tutte lo stesso tipo di dati oppure tipi di dati strettamente correlati e convertibili tra di loro.</span><span class="sxs-lookup"><span data-stu-id="9dc90-107">The expressions in a `Select`...`Case` construction should all be of the same data type or of closely related data types that can be converted to each other.</span></span> <span data-ttu-id="9dc90-108">Questo perché almeno un valore di ogni istruzione `Case` viene confrontato con l'espressione di test sulla quale si basa la costruzione `Select`...`Case` .</span><span class="sxs-lookup"><span data-stu-id="9dc90-108">This is because each `Case` statement compares at least one value against the test expression on which the `Select`...`Case` construction is based.</span></span>  
  
 <span data-ttu-id="9dc90-109">Per impostazione predefinita, si tratta di un messaggio di avviso.</span><span class="sxs-lookup"><span data-stu-id="9dc90-109">By default, this message is a warning.</span></span> <span data-ttu-id="9dc90-110">Per informazioni su come nascondere gli avvisi o considerarli come errori, vedere [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span><span class="sxs-lookup"><span data-stu-id="9dc90-110">For information on hiding warnings or treating warnings as errors, see [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span></span>
