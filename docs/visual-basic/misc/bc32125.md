# <a name="cannot-implement-39ltinterfacename1gtltmembernamegt39-because-its-implementation-could-conflict-with-the-implementation-for-39ltinterfacename2gtltmembernamegt39-for-some-type-arguments"></a><span data-ttu-id="5ee9e-102">Non è possibile implementare &#39; &lt;nomeinterfaccia1&gt;.&lt; MemberName&gt;&#39; perché la relativa implementazione potrebbe essere in conflitto con l'implementazione per &#39;&lt; nomeinterfaccia2&gt;.&lt; MemberName&gt;&#39; per alcuni argomenti di tipo</span><span class="sxs-lookup"><span data-stu-id="5ee9e-102">Cannot implement &#39;&lt;interfacename1&gt;.&lt;membername&gt;&#39; because its implementation could conflict with the implementation for &#39;&lt;interfacename2&gt;.&lt;membername&gt;&#39; for some type arguments</span></span>
<span data-ttu-id="5ee9e-103">Una classe implementa più di un'interfaccia generica, una delle quali eredita dall'altra, e le due implementazioni di un membro di interfaccia potrebbero essere in conflitto per determinati valori di argomenti di tipo.</span><span class="sxs-lookup"><span data-stu-id="5ee9e-103">A class implements more than one generic interface, one of which inherits from another, and two implementations of an interface member could conflict for certain values of type arguments.</span></span>  
  
 <span data-ttu-id="5ee9e-104">Le istruzioni seguenti possono generare questo errore.</span><span class="sxs-lookup"><span data-stu-id="5ee9e-104">The following statements can generate this error.</span></span>  
  
```  
Public Interface iFace1(Of t)  
    Sub testSub()  
End Interface  
Public Interface iFace2(Of u)  
    Inherits iFace1(Of u)  
End Interface  
Public Class testClass(Of y, z)  
    Implements iFace1(Of y), iFace2(Of z)  
    Public Sub testSuby() Implements iFace1(Of y).testSub  
    End Sub  
    Public Sub testSubz() Implements iFace1(Of z).testSub  
    End Sub  
End Class  
```  
  
 <span data-ttu-id="5ee9e-105">Dal momento che `iFace2` eredita da `iFace1` usando il relativo parametro di tipo (`u`), `testClass` implementerebbe due versioni di `iFace1.testSub` con firme identiche se lo stesso argomento di tipo venisse passato a `y` e `z`.</span><span class="sxs-lookup"><span data-stu-id="5ee9e-105">Because `iFace2` inherits from `iFace1` using its own type parameter (`u`), `testClass` would implement two versions of `iFace1.testSub` with identical signatures if the same type argument were passed to `y` and `z`.</span></span> <span data-ttu-id="5ee9e-106">In questo modo si produrrebbe un'ambiguità sulla versione a cui accedere.</span><span class="sxs-lookup"><span data-stu-id="5ee9e-106">This would produce an ambiguity about which version to access.</span></span>  
  
 <span data-ttu-id="5ee9e-107">**ID errore:** BC32125</span><span class="sxs-lookup"><span data-stu-id="5ee9e-107">**Error ID:** BC32125</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="5ee9e-108">Per correggere l'errore</span><span class="sxs-lookup"><span data-stu-id="5ee9e-108">To correct this error</span></span>  
  
-   <span data-ttu-id="5ee9e-109">Modificare la struttura di ereditarietà delle interfacce in modo che `iFace1` non possa essere implementata con due differenti argomenti di tipo.</span><span class="sxs-lookup"><span data-stu-id="5ee9e-109">Change the inheritance structure of the interfaces so that `iFace1` could not be implemented with two different type arguments.</span></span>  
  
     <span data-ttu-id="5ee9e-110">-oppure-</span><span class="sxs-lookup"><span data-stu-id="5ee9e-110">-or-</span></span>  
  
-   <span data-ttu-id="5ee9e-111">Rimuovere dall'istruzione `Implements` una delle interfacce che producono il conflitto di implementazione.</span><span class="sxs-lookup"><span data-stu-id="5ee9e-111">Remove from the `Implements` statement one of the interfaces resulting in the implementation conflict.</span></span>
