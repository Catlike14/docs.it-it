# <a name="type-parameter-39lttypeparameternamegt39-for-39ltgenericprocedurenamegt39-cannot-be-inferred"></a><span data-ttu-id="ebd24-102">Parametro di tipo &#39; &lt;nomeparametrotipo&gt;&#39; per &#39;&lt; nomeroutinegenerica&gt;&#39; non può essere dedotto</span><span class="sxs-lookup"><span data-stu-id="ebd24-102">Type parameter &#39;&lt;typeparametername&gt;&#39; for &#39;&lt;genericprocedurename&gt;&#39; cannot be inferred</span></span>
<span data-ttu-id="ebd24-103">È stata chiamata una routine generica senza fornire un elenco di argomenti di tipo e l'inferenza del tipo non riesce per uno degli argomenti di tipo.</span><span class="sxs-lookup"><span data-stu-id="ebd24-103">A generic procedure is called without supplying a type argument list, and type inference fails for one of the type arguments.</span></span>  
  
 <span data-ttu-id="ebd24-104">Di norma, quando si chiama una routine generica, si fornisce un argomento di tipo per ogni parametro di tipo definito dalla routine.</span><span class="sxs-lookup"><span data-stu-id="ebd24-104">When you call a generic procedure, you normally supply a type argument for each type parameter defined by the procedure.</span></span> <span data-ttu-id="ebd24-105">È però possibile omettere completamente l'elenco degli argomenti di tipo.</span><span class="sxs-lookup"><span data-stu-id="ebd24-105">However, you have the alternative of omitting the type argument list entirely.</span></span> <span data-ttu-id="ebd24-106">In questo caso, il compilatore prova a dedurre il tipo di ogni argomento di tipo dal contesto della chiamata.</span><span class="sxs-lookup"><span data-stu-id="ebd24-106">When you do this, the compiler attempts to infer the type of each type argument from the context of your call.</span></span> <span data-ttu-id="ebd24-107">Per altre informazioni, vedere "Inferenza di tipi" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span><span class="sxs-lookup"><span data-stu-id="ebd24-107">For more information, see "Type Inference" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span></span>  
  
 <span data-ttu-id="ebd24-108">Una delle possibili cause degli errore di inferenza del tipo è la mancata corrispondenza di dimensioni tra un parametro di tipo e il tipo chiamante.</span><span class="sxs-lookup"><span data-stu-id="ebd24-108">One possible cause of type inference failure is a mismatch of rank between a type parameter and the calling type.</span></span> <span data-ttu-id="ebd24-109">Questa condizione è illustrata nel codice che segue.</span><span class="sxs-lookup"><span data-stu-id="ebd24-109">The following code illustrates this.</span></span>  
  
```  
Public Sub displayLargest(Of t As IComparable)(ByVal arg() As t)  
    ' Insert code to find and display the largest element of arg().  
End Sub  
Public Sub callGenericSub()  
    Dim testValue As Integer  
    findLargest(testValue)  
    Dim testMatrix(,) As Integer  
    findLargest(testMatrix)  
End Sub  
```  
  
 <span data-ttu-id="ebd24-110">Nel codice precedente, le due chiamate a `findLargest` generano questo errore perché il parametro di tipo `t` richiede una matrice unidimensionale, mentre il compilatore deduce dalle chiamate che gli argomenti di tipo sono costituiti da una matrice scalare (`testValue`) e una matrice bidimensionale (`testMatrix`).</span><span class="sxs-lookup"><span data-stu-id="ebd24-110">In the preceding code, the two calls to `findLargest` both produce this error, because the type parameter `t` calls for a one-dimensional array, whereas the type arguments the compiler infers from the calls are a scalar (`testValue`) and a two-dimensional array (`testMatrix`).</span></span>  
  
 <span data-ttu-id="ebd24-111">**ID errore:** BC32050</span><span class="sxs-lookup"><span data-stu-id="ebd24-111">**Error ID:** BC32050</span></span>
