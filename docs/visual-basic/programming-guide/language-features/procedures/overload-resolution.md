# <a name="overload-resolution-visual-basic"></a><span data-ttu-id="e8734-102">Risoluzione dell'overload (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="e8734-102">Overload Resolution (Visual Basic)</span></span>
<span data-ttu-id="e8734-103">Quando il [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] viene rilevata una chiamata a una routine che viene definita in diverse versioni di overload, il compilatore deve decidere quale overload da chiamare.</span><span class="sxs-lookup"><span data-stu-id="e8734-103">When the [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] compiler encounters a call to a procedure that is defined in several overloaded versions, the compiler must decide which of the overloads to call.</span></span> <span data-ttu-id="e8734-104">A tale scopo, eseguire la procedura seguente:</span><span class="sxs-lookup"><span data-stu-id="e8734-104">It does this by performing the following steps:</span></span>  
  
1.  <span data-ttu-id="e8734-105">**Accessibilità.**</span><span class="sxs-lookup"><span data-stu-id="e8734-105">**Accessibility.**</span></span> <span data-ttu-id="e8734-106">Elimina tutti gli overload con un livello di accesso che impedisce la chiamata al codice chiamante.</span><span class="sxs-lookup"><span data-stu-id="e8734-106">It eliminates any overload with an access level that prevents the calling code from calling it.</span></span>  
  
2.  <span data-ttu-id="e8734-107">**Numero di parametri.**</span><span class="sxs-lookup"><span data-stu-id="e8734-107">**Number of Parameters.**</span></span> <span data-ttu-id="e8734-108">Eliminazione di eventuali overload che definisce un numero di parametri diverso rispetto a quello fornito nella chiamata.</span><span class="sxs-lookup"><span data-stu-id="e8734-108">It eliminates any overload that defines a different number of parameters than are supplied in the call.</span></span>  
  
3.  <span data-ttu-id="e8734-109">**Tipi di dati di parametro.**</span><span class="sxs-lookup"><span data-stu-id="e8734-109">**Parameter Data Types.**</span></span> <span data-ttu-id="e8734-110">Il compilatore assegna delle preferenze di metodi di istanza sui metodi di estensione.</span><span class="sxs-lookup"><span data-stu-id="e8734-110">The compiler gives instance methods preference over extension methods.</span></span> <span data-ttu-id="e8734-111">Se viene trovato alcun metodo di istanza che richiede solo di ampliamento per rispondere alla chiamata di procedura, vengono eliminati tutti i metodi di estensione e il compilatore continua con solo i candidati di metodo di istanza.</span><span class="sxs-lookup"><span data-stu-id="e8734-111">If any instance method is found that requires only widening conversions to match the procedure call, all extension methods are dropped and the compiler continues with only the instance method candidates.</span></span> <span data-ttu-id="e8734-112">Se tale metodo di istanza non viene trovato, ma continua con l'istanza sia metodi di estensione.</span><span class="sxs-lookup"><span data-stu-id="e8734-112">If no such instance method is found, it continues with both instance and extension methods.</span></span>  
  
     <span data-ttu-id="e8734-113">In questo passaggio, essa elimina alcun overload per il quale i tipi di dati degli argomenti di chiamata non possono essere convertiti per i tipi di parametro definiti nell'overload.</span><span class="sxs-lookup"><span data-stu-id="e8734-113">In this step, it eliminates any overload for which the data types of the calling arguments cannot be converted to the parameter types defined in the overload.</span></span>  
  
4.  <span data-ttu-id="e8734-114">**Conversioni di restrizione.**</span><span class="sxs-lookup"><span data-stu-id="e8734-114">**Narrowing Conversions.**</span></span> <span data-ttu-id="e8734-115">Eliminazione di eventuali overload che richiede una conversione dai tipi di argomento chiamante per i tipi di parametro definito.</span><span class="sxs-lookup"><span data-stu-id="e8734-115">It eliminates any overload that requires a narrowing conversion from the calling argument types to the defined parameter types.</span></span> <span data-ttu-id="e8734-116">È true se il controllo dei tipi di passare ([istruzione Option Strict](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) è `On` o `Off`.</span><span class="sxs-lookup"><span data-stu-id="e8734-116">This is true whether the type checking switch ([Option Strict Statement](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) is `On` or `Off`.</span></span>  
  
5.  <span data-ttu-id="e8734-117">**Widening minimi.**</span><span class="sxs-lookup"><span data-stu-id="e8734-117">**Least Widening.**</span></span> <span data-ttu-id="e8734-118">Il compilatore considera gli overload rimanenti nelle coppie.</span><span class="sxs-lookup"><span data-stu-id="e8734-118">The compiler considers the remaining overloads in pairs.</span></span> <span data-ttu-id="e8734-119">Per ogni coppia, confronta i tipi di dati dei parametri definiti.</span><span class="sxs-lookup"><span data-stu-id="e8734-119">For each pair, it compares the data types of the defined parameters.</span></span> <span data-ttu-id="e8734-120">Se i tipi in uno degli overload tutti vengono ampliati ai tipi corrispondenti negli altri, il compilatore elimina quest'ultimo.</span><span class="sxs-lookup"><span data-stu-id="e8734-120">If the types in one of the overloads all widen to the corresponding types in the other, the compiler eliminates the latter.</span></span> <span data-ttu-id="e8734-121">Vale a dire, viene mantenuto l'overload che richiede il minor grado di ampliamento.</span><span class="sxs-lookup"><span data-stu-id="e8734-121">That is, it retains the overload that requires the least amount of widening.</span></span>  
  
6.  <span data-ttu-id="e8734-122">**Singolo candidato.**</span><span class="sxs-lookup"><span data-stu-id="e8734-122">**Single Candidate.**</span></span> <span data-ttu-id="e8734-123">Vengono presi in considerazione gli overload nelle coppie finché l'unico overload rimane e viene risolta la chiamata all'overload.</span><span class="sxs-lookup"><span data-stu-id="e8734-123">It continues considering overloads in pairs until only one overload remains, and it resolves the call to that overload.</span></span> <span data-ttu-id="e8734-124">Se il compilatore non è possibile ridurre gli overload a uno solo, viene generato un errore.</span><span class="sxs-lookup"><span data-stu-id="e8734-124">If the compiler cannot reduce the overloads to a single candidate, it generates an error.</span></span>
