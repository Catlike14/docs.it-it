## <a name="reference-types"></a><span data-ttu-id="6613c-113">Tipi riferimento</span><span class="sxs-lookup"><span data-stu-id="6613c-113">Reference types</span></span>

```csharp
MyClass mc = new MyClass();
MyClass mc2 = mc;
```

<span data-ttu-id="6613c-116">Quando viene creato l'oggetto, la memoria viene allocata nell'heap gestito e la variabile mantiene solo un riferimento al percorso dell'oggetto.</span><span class="sxs-lookup"><span data-stu-id="6613c-116">When the object is created, the memory is allocated on the managed heap, and the variable holds only a reference to the location of the object.</span></span> <span data-ttu-id="6613c-117">I tipi nell'heap gestito richiedono un overhead quando vengono allocati e recuperati dalla funzionalità di gestione automatica della memoria di CLR, nota come *Garbage Collection*.</span><span class="sxs-lookup"><span data-stu-id="6613c-117">Types on the managed heap require overhead both when they are allocated and when they are reclaimed by the automatic memory management functionality of the CLR, which is known as *garbage collection*.</span></span> <span data-ttu-id="6613c-118">La Garbage Collection è tuttavia anche altamente ottimizzata e, nella maggior parte degli scenari, non causa un problema di prestazioni.</span><span class="sxs-lookup"><span data-stu-id="6613c-118">However, garbage collection is also highly optimized, and in most scenarios, it does not create a performance issue.</span></span> <span data-ttu-id="6613c-119">Per altre informazioni sulla Garbage Collection, vedere [Gestione automatica della memoria e Garbage Collection](../../../standard/garbage-collection/gc.md).</span><span class="sxs-lookup"><span data-stu-id="6613c-119">For more information about garbage collection, see [Automatic memory management and garbage collection](../../../standard/garbage-collection/gc.md).</span></span>

## <a name="creating-objects"></a><span data-ttu-id="6613c-127">Creazione di oggetti</span><span class="sxs-lookup"><span data-stu-id="6613c-127">Creating Objects</span></span>  
 <span data-ttu-id="6613c-128">Anche se vengono talvolta usati in modo intercambiabile, una classe e un oggetto sono elementi diversi.</span><span class="sxs-lookup"><span data-stu-id="6613c-128">Although they are sometimes used interchangeably, a class and an object are different things.</span></span> <span data-ttu-id="6613c-129">Una classe definisce un tipo di oggetto, ma non è un oggetto.</span><span class="sxs-lookup"><span data-stu-id="6613c-129">A class defines a type of object, but it is not an object itself.</span></span> <span data-ttu-id="6613c-130">Un oggetto è un'entità concreta ed è basato su una classe.  <span data-ttu-id="6613c-131">Gli oggetti possono essere creati tramite la parola chiave [new](../../../csharp/language-reference/keywords/new.md) seguita dal nome della classe su cui si baserà l'oggetto, nel modo seguente:</span><span class="sxs-lookup"><span data-stu-id="6613c-131">Objects can be created by using the [new](../../../csharp/language-reference/keywords/new.md) keyword followed by the name of the class that the object will be based on, like this:</span></span>

  ```csharp
  Customer object2;
  ```
  
 <span data-ttu-id="6613c-136">Non è consigliabile creare riferimenti a oggetti come questo che non fanno riferimento a un oggetto reale perché il tentativo di accedere a un oggetto tramite tale riferimento avrà esito negativo in fase di esecuzione.</span><span class="sxs-lookup"><span data-stu-id="6613c-136">We don't recommend creating object references such as this one that don't refer to an object because trying to access an object through such a reference will fail at run time.</span></span> <span data-ttu-id="6613c-137">Tuttavia, tale riferimento può essere creato per fare riferimento a un oggetto, creando un nuovo oggetto oppure assegnandolo a un oggetto esistente, come illustrato di seguito:</span><span class="sxs-lookup"><span data-stu-id="6613c-137">However, such a reference can be made to refer to an object, either by creating a new object, or by assigning it to an existing object, such as this:</span></span>  

 ```csharp
 Customer object3 = new Customer();
 Customer object4 = object3;
 ```
  
 <span data-ttu-id="6613c-138">Questo codice crea due riferimenti a oggetti che fanno entrambi riferimento allo stesso oggetto.</span><span class="sxs-lookup"><span data-stu-id="6613c-138">This code creates two object references that both refer to the same object.</span></span> <span data-ttu-id="6613c-139">Tutte le modifiche effettuate all'oggetto tramite `object3` si riflettono tuttavia nei successivi usi di `object4`.</span><span class="sxs-lookup"><span data-stu-id="6613c-139">Therefore, any changes to the object made through `object3` are reflected in subsequent uses of `object4`.</span></span> <span data-ttu-id="6613c-140">Poiché gli oggetti che si basano su classi vengono indicati tramite riferimenti, le classi sono note come tipi di riferimento.</span><span class="sxs-lookup"><span data-stu-id="6613c-140">Because objects that are based on classes are referred to by reference, classes are known as reference types.</span></span>  
  
## <a name="class-inheritance"></a><span data-ttu-id="6613c-141">Ereditarietà di classe</span><span class="sxs-lookup"><span data-stu-id="6613c-141">Class Inheritance</span></span>  

 <span data-ttu-id="6613c-142">Le classi supportano completamente l'*ereditarietà*, una caratteristica fondamentale nella programmazione orientata a oggetti.</span><span class="sxs-lookup"><span data-stu-id="6613c-142">Classes fully support *inheritance*, a fundamental characteristic of object-oriented programming.</span></span> <span data-ttu-id="6613c-143">Quando si crea una classe, è possibile ereditare da qualsiasi altra interfaccia o classe non definita come [sealed](../../../csharp/language-reference/keywords/sealed.md) e le altre classi possono ereditare dalla classe appena creata ed eseguire l'override dei metodi virtuali della classe.</span><span class="sxs-lookup"><span data-stu-id="6613c-143">When you create a class, you can inherit from any other interface or class that is not defined as [sealed](../../../csharp/language-reference/keywords/sealed.md), and other classes can inherit from your class and override class virtual methods.</span></span>

 <span data-ttu-id="6613c-144">L'ereditarietà si ottiene usando una *derivazione*, vale a dire che una classe viene dichiarata usando una *classe di base* da cui eredita dati e comportamento.</span><span class="sxs-lookup"><span data-stu-id="6613c-144">Inheritance is accomplished by using a *derivation*, which means a class is declared by using a *base class* from which it inherits data and behavior.</span></span> <span data-ttu-id="6613c-145">Una classe di base viene specificata tramite l'aggiunta di due punti e il nome della classe di base dopo il nome della classe derivata, nel modo seguente:</span><span class="sxs-lookup"><span data-stu-id="6613c-145">A base class is specified by appending a colon and the name of the base class following the derived class name, like this:</span></span>

 <span data-ttu-id="6613c-152">Una classe può essere dichiarata come [astratta](../../../csharp/language-reference/keywords/abstract.md).</span><span class="sxs-lookup"><span data-stu-id="6613c-152">A class can be declared [abstract](../../../csharp/language-reference/keywords/abstract.md).</span></span> <span data-ttu-id="6613c-153">Una classe astratta contiene metodi astratti che hanno una definizione di firma, ma senza implementazione.</span><span class="sxs-lookup"><span data-stu-id="6613c-153">An abstract class contains abstract methods that have a signature definition but no implementation.</span></span> <span data-ttu-id="6613c-154">Non è possibile creare un'istanza di classi astratte.</span><span class="sxs-lookup"><span data-stu-id="6613c-154">Abstract classes cannot be instantiated.</span></span> <span data-ttu-id="6613c-155">Le classi astratte possono essere usate solo tramite classi derivate che implementano i metodi astratti.</span><span class="sxs-lookup"><span data-stu-id="6613c-155">They can only be used through derived classes that implement the abstract methods.</span></span> <span data-ttu-id="6613c-156">Al contrario, una classe [sealed](../../../csharp/language-reference/keywords/sealed.md) non consente ad altre classi di derivare da tale classe.</span><span class="sxs-lookup"><span data-stu-id="6613c-156">By contrast, a [sealed](../../../csharp/language-reference/keywords/sealed.md) class does not allow other classes to derive from it.</span></span> <span data-ttu-id="6613c-157">Per altre informazioni, vedere [Classi e membri delle classi astratte e sealed](../../../csharp/programming-guide/classes-and-structs/abstract-and-sealed-classes-and-class-members.md).</span><span class="sxs-lookup"><span data-stu-id="6613c-157">For more information, see [Abstract and Sealed Classes and Class Members](../../../csharp/programming-guide/classes-and-structs/abstract-and-sealed-classes-and-class-members.md).</span></span>  
  
 <span data-ttu-id="6613c-158">Le definizioni di classe possono essere suddivise tra file di origine diversa.</span><span class="sxs-lookup"><span data-stu-id="6613c-158">Class definitions can be split between different source files.</span></span> <span data-ttu-id="6613c-159">Per altre informazioni, vedere [Classi e metodi parziali](../../../csharp/programming-guide/classes-and-structs/partial-classes-and-methods.md).</span><span class="sxs-lookup"><span data-stu-id="6613c-159">For more information, see [Partial Classes and Methods](../../../csharp/programming-guide/classes-and-structs/partial-classes-and-methods.md).</span></span>
