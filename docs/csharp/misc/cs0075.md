# <a name="compiler-error-cs0075"></a><span data-ttu-id="833b6-102">Errore del compilatore CS0075</span><span class="sxs-lookup"><span data-stu-id="833b6-102">Compiler Error CS0075</span></span>
<span data-ttu-id="833b6-103">Per eseguire il cast di un valore negativo, è necessario racchiuderlo tra parentesi</span><span class="sxs-lookup"><span data-stu-id="833b6-103">To cast a negative value, you must enclose the value in parentheses</span></span>  
  
 <span data-ttu-id="833b6-104">Se si esegue il cast usando una parola chiave che identifica un tipo predefinito, le parentesi non sono necessarie.</span><span class="sxs-lookup"><span data-stu-id="833b6-104">If you are casting using a keyword that identifies a predefined type, then you do not need parentheses.</span></span> <span data-ttu-id="833b6-105">In caso contrario, è necessario inserire le parentesi, altrimenti (x) –y non verrà considerata un'espressione cast.</span><span class="sxs-lookup"><span data-stu-id="833b6-105">Otherwise, you must put the parentheses because (x) –y will not be considered a cast expression.</span></span> <span data-ttu-id="833b6-106">Dalla Specifica del linguaggio C#, sezione 7.6.6:</span><span class="sxs-lookup"><span data-stu-id="833b6-106">From the C# Specification, Section 7.6.6:</span></span>  
  
 <span data-ttu-id="833b6-107">*Dalla regola di risoluzione dell'ambiguità segue che, se x e y sono identificatori, (x) y, (y, e (saranno sono espressioni cast, ma non (x)-y non lo è, anche se x identifica un tipo. Tuttavia, se x è una parola chiave che identifica un tipo predefinito (ad esempio, int), quindi il tutti e quattro le forme sono espressioni cast (perché tale parola chiave non può costituire un'espressione da solo).*</span><span class="sxs-lookup"><span data-stu-id="833b6-107">*From the disambiguation rule it follows that, if x and y are identifiers, (x)y, (x)(y), and (x)(-y) are cast-expressions, but (x)-y is not, even if x identifies a type. However, if x is a keyword that identifies a predefined type (such as int), then all four forms are cast-expressions (because such a keyword could not possibly be an expression by itself).*</span></span>  
  
 <span data-ttu-id="833b6-108">Il codice seguente genera l'errore CS0075:</span><span class="sxs-lookup"><span data-stu-id="833b6-108">The following code generates CS0075:</span></span>  
  
```  
// CS0075  
namespace MyNamespace  
{  
    enum MyEnum { }  
    public class MyClass  
    {  
        public static void Main()  
        {  
            // To fix the error, place the negative  
            // values below in parentheses  
            int i = (System.Int32) - 4; //CS0075  
            MyEnum e = (MyEnum) - 1;    //CS0075  
            System.Console.WriteLine(i); //to avoid warning  
            System.Console.WriteLine(e); //to avoid warning  
        }  
    }  
}  
```
