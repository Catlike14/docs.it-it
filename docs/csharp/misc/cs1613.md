# <a name="compiler-error-cs1613"></a><span data-ttu-id="d1f16-102">Errore del compilatore CS1613</span><span class="sxs-lookup"><span data-stu-id="d1f16-102">Compiler Error CS1613</span></span>
<span data-ttu-id="d1f16-103">La classe wrapper 'class' della coclasse gestita per l'interfaccia 'interface' non è stata trovata. Probabilmente manca un riferimento all'assembly</span><span class="sxs-lookup"><span data-stu-id="d1f16-103">The managed coclass wrapper class 'class' for interface 'interface' cannot be found (are you missing an assembly reference?)</span></span>  
  
 <span data-ttu-id="d1f16-104">Si è provato a creare un'istanza di un oggetto COM da un'interfaccia.</span><span class="sxs-lookup"><span data-stu-id="d1f16-104">An attempt was made to instantiate a COM object from an interface.</span></span> <span data-ttu-id="d1f16-105">L'interfaccia ha gli attributi **ComImport** e `CoClass` , ma il compilatore non riesce a trovare il tipo specificato per l'attributo `CoClass` .</span><span class="sxs-lookup"><span data-stu-id="d1f16-105">The interface has the **ComImport** and `CoClass` attributes, but the compiler cannot find the type given for the `CoClass` attribute.</span></span>  
  
 <span data-ttu-id="d1f16-106">Per correggere l'errore, provare una delle seguenti soluzioni:</span><span class="sxs-lookup"><span data-stu-id="d1f16-106">To resolve this error, you can try one of the following:</span></span>  
  
-   <span data-ttu-id="d1f16-107">Aggiungere un riferimento all'assembly che abbia la coclasse. L'interfaccia e la coclasse devono trovarsi quasi sempre nello stesso assembly.</span><span class="sxs-lookup"><span data-stu-id="d1f16-107">Add a reference to the assembly that has the coclass (most of the time the interface and coclass should be in the same assembly).</span></span> <span data-ttu-id="d1f16-108">Per altre informazioni, vedere [/reference](../../csharp/language-reference/compiler-options/reference-compiler-option.md) o [Finestra di dialogo Aggiungi riferimento](http://msdn.microsoft.com/en-us/2feb0fe2-0805-4cc9-8cba-b0315849dfb7) .</span><span class="sxs-lookup"><span data-stu-id="d1f16-108">See [/reference](../../csharp/language-reference/compiler-options/reference-compiler-option.md) or [Add Reference Dialog Box](http://msdn.microsoft.com/en-us/2feb0fe2-0805-4cc9-8cba-b0315849dfb7) for information.</span></span>  
  
-   <span data-ttu-id="d1f16-109">Correggere l'attributo `CoClass` nell'interfaccia.</span><span class="sxs-lookup"><span data-stu-id="d1f16-109">Fix the `CoClass` attribute on the interface.</span></span>  
  
 <span data-ttu-id="d1f16-110">L'esempio che segue illustra l'uso corretto di **CoClassAttribute**:</span><span class="sxs-lookup"><span data-stu-id="d1f16-110">The following sample demonstrates correct usage of **CoClassAttribute**:</span></span>  
  
```  
// CS1613.cs  
using System;  
using System.Runtime.InteropServices;  
  
[Guid("1FFD7840-E82D-4268-875C-80A160C23296")]  
[ComImport()]  
[CoClass(typeof(A))]  
public interface IA{}  
public class A : IA {}  
  
public class AA  
{  
   public static void Main()  
   {  
      IA i;  
      i = new IA(); // This is equivalent to new A().  
                    // because of the CoClass attribute on IA  
   }  
}  
```
