# <a name="compiler-error-cs1614"></a><span data-ttu-id="8c444-102">Errore del compilatore CS1614</span><span class="sxs-lookup"><span data-stu-id="8c444-102">Compiler Error CS1614</span></span>
<span data-ttu-id="8c444-103">'nome' è ambiguo tra 'attributo1' e 'attributo2'.</span><span class="sxs-lookup"><span data-stu-id="8c444-103">'name' is ambiguous; between 'attribute1' and 'attribute2'.</span></span> <span data-ttu-id="8c444-104">Utilizzare '@attribute' o 'attributoAttribute'</span><span class="sxs-lookup"><span data-stu-id="8c444-104">use either '@attribute' or 'attributeAttribute'</span></span>  
  
 <span data-ttu-id="8c444-105">Il compilatore ha rilevato una specifica di attributo ambigua.</span><span class="sxs-lookup"><span data-stu-id="8c444-105">The compiler has encountered an ambiguous attribute specification.</span></span>  
  
 <span data-ttu-id="8c444-106">Per comodità, il compilatore C# consente di specificare **ExampleAttribute** semplicemente come `[Example]`.</span><span class="sxs-lookup"><span data-stu-id="8c444-106">For convenience, the C# compiler allows you to specify **ExampleAttribute** as just `[Example]`.</span></span> <span data-ttu-id="8c444-107">Ciò comporta tuttavia una situazione di ambiguità se esiste una classe Attribute denominata `Example` insieme a **ExampleAttribute**, perché il compilatore non è in grado di stabilire se `[Example]` fa riferimento all'attributo `Example` o all'attributo **ExampleAttribute**.</span><span class="sxs-lookup"><span data-stu-id="8c444-107">However, ambiguity arises if an attribute class named `Example` exists along with **ExampleAttribute**, because the compiler cannot tell if `[Example]` refers to the `Example` attribute or the **ExampleAttribute** attribute.</span></span> <span data-ttu-id="8c444-108">Per chiarire, usare `[@Example]` per l'attributo `Example` e `[ExampleAttribute]` per **ExampleAttribute**.</span><span class="sxs-lookup"><span data-stu-id="8c444-108">To clarify, use `[@Example]` for the `Example` attribute and `[ExampleAttribute]` for **ExampleAttribute**.</span></span>  
  
 <span data-ttu-id="8c444-109">L'esempio seguente genera l'errore CS1614:</span><span class="sxs-lookup"><span data-stu-id="8c444-109">The following sample generates CS1614:</span></span>  
  
```csharp  
// CS1614.cs  
using System;  
  
// Both of the following classes are valid attributes with valid  
// names (MySpecial and MySpecialAttribute). However, because the lookup  
// rules for attributes involves auto-appending the 'Attribute' suffix  
// to the identifier, these two attributes become ambiguous; that is,  
// if you specify MySpecial, the compiler can't tell if you want  
// MySpecial or MySpecialAttribute.  
  
public class MySpecial : Attribute {  
   public MySpecial() {}  
}  
  
public class MySpecialAttribute : Attribute {  
   public MySpecialAttribute() {}  
}  
  
class MakeAWarning {  
   [MySpecial()] // CS1614  
                 // Ambiguous: MySpecial or MySpecialAttribute?  
   public static void Main() {  
   }  
  
   [@MySpecial()] // This isn't ambiguous, it binds to the first attribute above.  
   public static void NoWarning() {  
   }  
  
   [MySpecialAttribute()] // This isn't ambiguous, it binds to the second attribute above.  
   public static void NoWarning2() {  
   }  
  
   [@MySpecialAttribute()] // This is also legal.  
   public static void NoWarning3() {  
   }  
}  
```
