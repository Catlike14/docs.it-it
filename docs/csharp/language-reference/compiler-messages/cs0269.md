# <a name="compiler-error-cs0269"></a><span data-ttu-id="519fc-102">Errore del compilatore CS0269</span><span class="sxs-lookup"><span data-stu-id="519fc-102">Compiler Error CS0269</span></span>
<span data-ttu-id="519fc-103">Uso del parametro out "parameter" non assegnato</span><span class="sxs-lookup"><span data-stu-id="519fc-103">Use of unassigned out parameter 'parameter'</span></span>  
  
 <span data-ttu-id="519fc-104">Il compilatore non verifica se al parametro out è stato assegnato un valore prima dell'uso e il valore può non essere definito quando viene assegnato.</span><span class="sxs-lookup"><span data-stu-id="519fc-104">The compiler could not verify that the out parameter was assigned a value before it was used; its value may be undefined when assigned.</span></span> <span data-ttu-id="519fc-105">Verificare che sia assegnato un valore ai parametri `out` nel metodo chiamato prima di accedere al valore.</span><span class="sxs-lookup"><span data-stu-id="519fc-105">Be sure to assign a value to `out` parameters in the called method before accessing the value.</span></span> <span data-ttu-id="519fc-106">Se è necessario usare il valore della variabile passata, usare un parametro `ref`.</span><span class="sxs-lookup"><span data-stu-id="519fc-106">If you need to use the value of the variable passed in, use a `ref` parameter instead.</span></span> <span data-ttu-id="519fc-107">Per altre informazioni, vedere [Passaggio di parametri](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span><span class="sxs-lookup"><span data-stu-id="519fc-107">For more information, see [Passing Parameters](../../../csharp/programming-guide/classes-and-structs/passing-parameters.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="519fc-108">Esempio</span><span class="sxs-lookup"><span data-stu-id="519fc-108">Example</span></span>  
 <span data-ttu-id="519fc-109">L'esempio seguente genera l'errore CS0269:</span><span class="sxs-lookup"><span data-stu-id="519fc-109">The following sample generates CS0269:</span></span>  
  
```csharp  
// CS0269.cs  
class C  
{  
    public static void F(out int i)  
    // One way to resolve the error is to use a ref parameter instead  
    // of an out parameter.  
    //public static void F(ref int i)  
    {  
        // The following line causes a compiler error because no value  
        // has been assigned to i.  
        int k = i;  // CS0269  
        i = 1;  
        // The error does not occur if the order of the two previous   
        // lines is reversed.  
    }  
  
    public static void Main()  
    {  
        int myInt = 1;  
        F(out myInt);  
        // If the declaration of method F is changed to require a ref  
        // parameter, ref must be specified in the call as well.  
        //F(ref myInt);  
    }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="519fc-110">Esempio</span><span class="sxs-lookup"><span data-stu-id="519fc-110">Example</span></span>  
 <span data-ttu-id="519fc-111">Ciò può verificarsi anche se l'inizializzazione di una variabile avviene in un blocco try e il compilatore non è in grado di verificare se verrà completata:</span><span class="sxs-lookup"><span data-stu-id="519fc-111">This could also occur if the initialization of a variable occurs in a try block, which the compiler is unable to verify will execute successfully:</span></span>  
  
```csharp  
// CS0269b.cs  
class C  
{  
    public static void F(out int i)  
    {  
        try  
        {  
            // Assignment occurs, but compiler can't verify it  
            i = 1;  
        }  
        catch  
        {  
        }  
  
        int k = i;  // CS0269  
        i = 1;  
    }  
  
    public static void Main()  
    {  
        int myInt;  
        F(out myInt);  
    }  
}  
```
