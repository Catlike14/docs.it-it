# <a name="compiler-error-cs0103"></a><span data-ttu-id="721cd-102">Errore del compilatore CS0103</span><span class="sxs-lookup"><span data-stu-id="721cd-102">Compiler Error CS0103</span></span>
<span data-ttu-id="721cd-103">Il nome "identifier" non esiste nel contesto corrente</span><span class="sxs-lookup"><span data-stu-id="721cd-103">The name 'identifier' does not exist in the current context</span></span>  
  
 <span data-ttu-id="721cd-104">Si è tentato di usare un nome che non esiste nella classe, nello [spazio dei nomi](../../../csharp/language-reference/keywords/namespace.md) o nell'ambito.</span><span class="sxs-lookup"><span data-stu-id="721cd-104">An attempt was made to use a name that does not exist in the class, [namespace](../../../csharp/language-reference/keywords/namespace.md), or scope.</span></span> <span data-ttu-id="721cd-105">Controllare l'ortografia del nome, le direttive using e i riferimenti all'assembly per verificare la disponibilità del nome che si tenta di utilizzare.</span><span class="sxs-lookup"><span data-stu-id="721cd-105">Check the spelling of the name and check your using directives and assembly references to make sure that the name that you are trying to use is available.</span></span>  
  
 <span data-ttu-id="721cd-106">Questo errore si verifica frequentemente quando si dichiara una variabile all'interno di un ciclo o di un blocco `try` o `if` e successivamente si tenta di accedere alla variabile dal blocco di codice che la contiene o da un altro blocco di codice, come mostrato nell'esempio riportato di seguito.</span><span class="sxs-lookup"><span data-stu-id="721cd-106">This error frequently occurs if you declare a variable in a loop or a `try` or `if` block and then attempt to access it from an enclosing code block or a separate code block, as shown in the following example.</span></span>  
  
```csharp  
using System;  
  
class MyClass1  
{  
    public static void Main()  
    {  
        try  
        {  
            // The following declaration is only available inside the try block.  
            MyClass1 conn = new MyClass1();  
        }  
        catch (Exception e)  
        {  
            // The following expression causes error CS0103, because variable  
            // conn only exists in the try block.  
            if (conn != null)   
                Console.WriteLine("{0}", e);  
        }  
    }  
}  
```  
  
 <span data-ttu-id="721cd-107">Nell'esempio seguente l'errore viene risolto.</span><span class="sxs-lookup"><span data-stu-id="721cd-107">The following example resolves the error.</span></span>  
  
```csharp  
using System;  
  
class MyClass2  
{  
    public static void Main()  
    {  
        // To resolve the error in the example, the first step is to   
        // move the declaration of conn out of the try block. The following  
        // declaration is available throughout the Main method.  
        MyClass2 conn = null;  
        try  
        {  
            // Inside the try block, use the conn variable that you declared  
            // previously.  
            conn = new MyClass2();  
        }  
        catch (Exception e)  
        {  
            // The following expression no longer causes an error, because  
            // the declaration of conn is in scope.  
            if (conn != null)   
                Console.WriteLine("{0}", e);  
        }  
    }  
}  
```
