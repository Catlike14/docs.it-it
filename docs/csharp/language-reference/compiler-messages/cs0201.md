# <a name="compiler-error-cs0201"></a>Errore del compilatore CS0201
Si possono usare come istruzioni solo le assegnazioni, le chiamate, gli incrementi, i decrementi e le nuove espressioni dell'oggetto  
  
 Il compilatore genera un errore quando rileva un'istruzione non valida. Un'istruzione non valida è qualsiasi riga o serie di righe che termina con un punto e virgola che non rappresenta un'operazione di assegnazione ([=](../../../csharp/language-reference/operators/assignment-operator.md)), chiamata di metodo [()](../../../csharp/language-reference/operators/invocation-operator.md), [new](../../../csharp/language-reference/keywords/new.md), [--](../../../csharp/language-reference/operators/decrement-operator.md) o [++](../../../csharp/language-reference/operators/increment-operator.md). Per altre informazioni, vedere [Istruzioni, espressioni e operatori](../../../csharp/programming-guide/statements-expressions-operators/index.md).  
  
## <a name="example"></a>Esempio  
 L'esempio seguente genera l'errore CS0201 perché 2 * 3 è un'espressione e non un'istruzione. Per fare in modo che il codice venga compilato, provare ad assegnare il valore dell'espressione a una variabile.  
  
```csharp  
// CS0201.cs  
public class MainClass  
{  
   public static void Main()  
   {  
      2 * 3;   // CS0201  
      // Try the following line instead.  
      //   int i = 2 * 3;  
   }  
}  
```  
  
## <a name="example"></a>Esempio  
 L'esempio seguente genera l'errore CS0201 perché checked da solo non è un'istruzione, anche se è parametrizzato da un'operazione di incremento.  
  
```csharp  
// CS0201_b.cs  
// compile with: /target:library  
public class MyList<T>   
{  
   public void Add(T x)  
   {  
      int i = 0;  
      if ( (object)x == null)  
      {  
         checked(i++);   // CS0201  
  
         // OK  
         checked {  
            i++;   
         }  
      }  
   }  
}  
```
