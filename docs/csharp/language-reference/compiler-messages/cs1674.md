# <a name="compiler-error-cs1674"></a>Errore del compilatore CS1674
'T': il tipo usato in un'istruzione using deve essere convertibile in modo implicito in 'System.IDisposable'  
  
 L'[istruzione using](../../../csharp/language-reference/keywords/using-statement.md) è progettata per garantire l'eliminazione di un oggetto alla fine del blocco `using`, di conseguenza, solo i tipi eliminabili possono essere usati in un'istruzione di questo tipo. Ad esempio, i tipi valore non sono eliminabili e i parametri di tipo non vincolati come classi non possono essere considerati eliminabili.  
  
## <a name="example"></a>Esempio  
 L'esempio seguente genera l'errore CS1674.  
  
```csharp  
// CS1674.cs  
class C  
{  
   public static void Main()  
   {  
      int a = 0;  
      a++;  
  
      using (a) {}   // CS1674  
   }  
}  
```  
  
## <a name="example"></a>Esempio  
 L'esempio seguente genera l'errore CS1674.  
  
```csharp  
// CS1674_b.cs  
using System;  
class C {  
   public void Test() {  
      using (C c = new C()) {}   // CS1674  
   }  
}  
  
// OK  
class D : IDisposable {  
   void IDisposable.Dispose() {}  
   public void Dispose() {}  
  
   public static void Main() {  
      using (D d = new D()) {}  
   }  
}  
```  
  
## <a name="example"></a>Esempio  
 Il caso seguente illustra la necessità di un vincolo di tipo di classe per garantire che un parametro di tipo sconosciuto sia eliminabile. L'esempio seguente genera l'errore CS1674.  
  
```csharp  
// CS1674_c.cs  
// compile with: /target:library  
using System;  
public class C<T>  
// Add a class type constraint that specifies a disposable class.  
// Uncomment the following line to resolve.  
// public class C<T> where T : IDisposable  
{  
   public void F(T t)  
   {  
      using (t) {}   // CS1674  
   }  
}  
```
