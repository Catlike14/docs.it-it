# <a name="compiler-warning-level-2-cs0467"></a>Avviso del compilatore (livello 2) CS0467
Ambiguità tra il metodo "method" e il non metodo "non-method" usando il gruppo di metodi.  
  
 L'errore di ambiguità è causato dalla presenza di membri ereditati da interfacce diverse che hanno la stessa firma.  
  
## <a name="example"></a>Esempio  
 L'esempio seguente genera l'errore CS0467.  
  
```csharp  
// CS0467.cs  
interface IList   
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver   
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);     
        // To resolve the warning, you can change the name of the method or   
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()   
    {  
    }  
}  
```
