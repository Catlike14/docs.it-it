## <a name="self-contained-deployments-scd"></a><span data-ttu-id="fbe2f-132">Distribuzioni autonome</span><span class="sxs-lookup"><span data-stu-id="fbe2f-132">Self-contained deployments (SCD)</span></span>

<span data-ttu-id="fbe2f-133">Per una distribuzione autonoma, si distribuiscono l'app e le eventuali dipendenze di terze parti richieste, insieme alla versione di .NET Core usata per compilare l'app.</span><span class="sxs-lookup"><span data-stu-id="fbe2f-133">For a self-contained deployment, you deploy your app and any required third-party dependencies along with the version of .NET Core that you used to build the app.</span></span> <span data-ttu-id="fbe2f-134">Poiché la creazione di una distribuzione autonoma non include le [dipendenze native di .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) su più piattaforme, queste dipendenze devono essere presenti prima di eseguire l'applicazione.</span><span class="sxs-lookup"><span data-stu-id="fbe2f-134">Creating an SCD doesn't include the [native dependencies of .NET Core](https://github.com/dotnet/core/blob/master/Documentation/prereqs.md) on various platforms, so these must be present before the app runs.</span></span>

<span data-ttu-id="fbe2f-135">Le distribuzioni dipendenti dal framework e le distribuzioni autonome usano eseguibili host separati, quindi è possibile firmare un eseguibile host per una distribuzione autonoma con la firma del server di pubblicazione.</span><span class="sxs-lookup"><span data-stu-id="fbe2f-135">FDD and SCD deployments use separate host executables, so you can sign a host executable for an SCD with your publisher signature.</span></span>

<span data-ttu-id="fbe2f-137">Una distribuzione autonoma offre due vantaggi principali:</span><span class="sxs-lookup"><span data-stu-id="fbe2f-137">Deploying a Self-contained deployment has two major advantages:</span></span>

- <span data-ttu-id="fbe2f-138">Si dispone del controllo esclusivo della versione di .NET Core che viene distribuita con l'app.</span><span class="sxs-lookup"><span data-stu-id="fbe2f-138">You have sole control of the version of .NET Core that is deployed with your app.</span></span> <span data-ttu-id="fbe2f-139">.NET Core può essere usato solo dall'utente.</span><span class="sxs-lookup"><span data-stu-id="fbe2f-139">.NET Core can be serviced only by you.</span></span>

- <span data-ttu-id="fbe2f-140">È possibile garantire che il sistema di destinazione esegua l'app .NET Core, poiché viene specificata la versione di .NET Core in cui verrà eseguita.</span><span class="sxs-lookup"><span data-stu-id="fbe2f-140">You can be assured that the target system can run your .NET Core app, since you're providing the version of .NET Core that it will run on.</span></span>

<span data-ttu-id="fbe2f-141">Sono presenti anche alcuni svantaggi:</span><span class="sxs-lookup"><span data-stu-id="fbe2f-141">It also has a number of disadvantages:</span></span>
